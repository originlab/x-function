メニュー情報

データ整形: 列アンスタッキング

概要

グループ列を使用してデータをアンスタック

追加の情報

必要なOriginのバージョン：Origin 8.1 SR0以上。Origin 8.5.1更新。

コマンドラインでの使用法

1. wunstackcol irng1:=2 irng2:=1 missing:=1;

2. wunstackcol irng1:=col(2) irng2:=col(1) nonstack:=1 other:=col(3);

3. wunstackcol -r 2 irng1:=(3:5) irng2:=col(2) sort:=data pos:=udl label:=L;

スクリプトからXファンクションにアクセスする場合、追加のオプションスイッチについてのページを参照してください。

オプションリスト:

  • longest: 最長グループ一致
    このオプションは、グループ列と他の列でアンスタックデータの構造が同じ場合に使用されます。このオプションを使用して列をアンスタックする前に、データは最初にグループ列でソートされ、次に他の列でソートされることに注意してください。
  • combine: 全てのグループを統合する
    グループ列のデータと他の列のデータを結合し、インデックスとして使用してアンスタックを行います。
欠損データを単一グループとして含める missing

出力列のソート sort

入力

int

0
 出力列の並べ替え方法を指定します。

オプションリスト:

  • group: グループ変数
    出力列を、グループ変数の値でアルファベット順に並べます。例えば、グループ変数「Make」に "Acura" と "Buick" がある場合、アンスタック後は「Acura」列が先、「Buick」列が後に並びます。
  • data: アンスタックされるデータ
    アンスタックされるデータの順序を保持します。たとえば、アンスタックされるデータ = [Book1]Sheet1!(3,4) の場合、列3のデータ列が先に、列4のデータ列がその後に表示されます。逆にアンスタックされるデータ = [Book1]Sheet1!(4,3) の場合は、列4、列3の順になります。

グループ情報名の引用元 label

入力

string

<unassigned>
 このドロップダウンリストは、グループ情報の出力先としてユーザ定義パラメータが設定されている場合にのみ使用できます。グループ情報を格納するユーザ定義パラメータ行の名前付けに使用されます。ソースワークシートのグループ列のラベル行から名前を付けます。選択したラベル行には、ユーザ定義パラメータ行の名前用の文字列が必要で、ない場合はエラーメッセージが表示されます。ラベル行の指定については、列ラベル行文字を参照してください。
